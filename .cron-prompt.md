# ExcelMCP — 3 节点精简迭代（时间驱动）

> 不读 SKILL.md。只循环执行 3 个节点，时间不够就停。

## 开始前
exec: `date +%s > /root/.openclaw/workspace/excel-mcp-server/.iterate-start`
这记录本轮开始时间。

## 循环

### ① CHECK（5秒）
exec: `cd /root/.openclaw/workspace/excel-mcp-server && grep -cE '\- \[OPEN\]|状态.*: OPEN' REQUIREMENTS.md`
- 0 → 推送"无需求"结束
- >0 → 继续

### ② CODE（每个REQ约2-3分钟）
sessions_spawn(runtime:"acp", agentId:"claude", mode:"run", runTimeoutSeconds:300, cwd:"/root/.openclaw/workspace/excel-mcp-server", task:"读REQUIREMENTS.md，完成下一个OPEN的P0 REQ（按编号顺序），标DONE，git commit。完成编码后跑pytest tests/ -x -q确认不破。不自造需求。只做1个。")

等结果返回。

### ③ VERIFY + 时间判断（30秒）
exec: `cd /root/.openclaw/workspace/excel-mcp-server && git log --oneline -1 && echo "---" && echo $(( 1775783695 - $(cat .iterate-start) ))`
- 新 commit 包含 REQ → ✅ 这轮完成
- 无新 commit → ❌ 推送"编码失败"结束

**时间判断**：
- 已用时间 < 10分钟（600秒）→ 还有时间，回到 ① 做下一个
- 已用时间 ≥ 10分钟 → 时间不够了，推送汇总报告结束

## 停止条件
- 无 OPEN 需求
- 已用 ≥ 10分钟
- 连续 2 次无产出
- 已做 ≥ 8 个 REQ

## 推送报告格式（🔴 必须严格遵守）

### 有产出的报告
推送前用 exec 获取以下信息：
```bash
cd /root/.openclaw/workspace/excel-mcp-server
echo "=== COMMITS ==="
git log --oneline --since="$(date -d @$(cat .iterate-start) +%Y-%m-%dT%H:%M:%S)" 2>/dev/null || git log --oneline -5
echo "=== OPEN_COUNT ==="
grep -cE '\- \[OPEN\]|状态.*: OPEN' REQUIREMENTS.md
echo "=== DONE_COUNT ==="
grep -cE '\- \[DONE\]|状态.*: DONE' REQUIREMENTS.md
echo "=== LAST_REQ ==="
grep -B2 -E '\- \[DONE\]|状态.*: DONE' REQUIREMENTS.md | tail -6
echo "=== NEXT_OPEN ==="
grep -B2 -E '\- \[OPEN\]|状态.*: OPEN' REQUIREMENTS.md | head -3
```

然后推送以下格式到飞书和QQ：

```

• commit_msg1 (REQ-XXX)

🔜 下一个: REQ-XXX 简述

📝 最近commit:
• hash1 msg1
• hash2 msg2
```
