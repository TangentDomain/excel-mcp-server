## 第4步：链式执行——消除轮次间空等（REQ-029）
**链式执行机制**：当还有OPEN需求且本轮有进展时，连续执行下一轮，最多连续5轮
- 检查是否有OPEN任务：`grep -r "状态.*OPEN" REQUIREMENTS.md || echo "无OPEN任务"`
- 检查本轮是否有进展：`python3 scripts/round-snapshot.py read` 获取上轮快照，对比files_modified和commits
- **连续执行逻辑**：
  - 如果有OPEN任务且已连续执行轮次 < 5 → 递增轮次号，回到第2步继续执行
  - 如果无OPEN任务 或 已连续执行5轮 → 结束链式执行，进入第5步
  - ⚠️ 注意：只要有OPEN需求就继续，不要求本轮一定有代码变更。分析、重构、文档修复都是进展。
- **连续执行限制**：最多连续5轮，达到上限后强制结束
- **轮次递增**：`round=$((round + 1))`，每轮都写入新快照

## 第5步：轮次快照写入（REQ-022）
**轮次快照机制**：记录本轮进展，用于下一轮追溯
- exec：`git log --oneline -1` 获取最新commit
- exec：`git diff --stat` 获取文件变更统计
- exec：`python3 scripts/round-snapshot.py write {当前轮次} REQ-022 "K3完成" "{文件变更列表}" "{commit列表}" "测试通过" "K0" --next-step "K0"`
- 记录快照完成：`echo "[$(date +%H:%M:%S)] 轮次快照写入完成" >> .claude-progress.md`
