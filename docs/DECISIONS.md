## 2026-03-27 | 分支策略优化
- **决策**：使用git worktree管理分支，避免分支冲突
- **原因**：传统分支切换容易出错，需要更安全的分支管理
- **工作流程**：
  - 每个需求开独立feature分支+worktree
  - 一次只开一个worktree，完成清理后再开下一个
  - 测试通过→合develop→合main→清理worktree
- **失败处理**：做不好→丢弃worktree，不影响develop
- **效果**：分支管理更安全，失败不影响主干

## 2026-03-27 | 发布流程标准化
- **决策**：建立标准化的PyPI发布流程，确保发布质量
- **原因**：发布需要质量保证，避免有问题的版本上线
- **发布步骤**：
  1. 版本号更新：pyproject.toml + __init__.py
  2. 构建：uv build
  3. 全量测试：pytest -q -n auto
  4. 本地测试：pip install + 启动测试
  5. 发布：uv publish
  6. 远程验证：pip install --dry-run
  7. tag+push：git tag + git push
- **版本策略**：功能→minor，bug修复→patch
- **效果**：发布流程标准化，质量可控

## 2026-03-27 | 测试策略优化
- **决策**：优化测试策略，提高开发效率
- **原因**：全量测试耗时，需要优化日常开发体验
- **开发中**：只跑受影响的测试文件（3-5秒）
- **全量测试两个时机**：
  - 第一轮评估基线
  - 合并main前
- **并行优化**：必须用pytest-xdist -n auto
- **超时保护**：pytest --timeout=30防止单个测试卡住
- **效果**：开发效率提升，质量保证不变
