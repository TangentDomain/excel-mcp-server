## D001: 2025-12-01 初始架构设计
**需求**: 项目启动，基础架构搭建
**问题**: 从零开始构建Excel MCP服务器，需要清晰的模块架构
**根因**: Excel操作涉及多种复杂场景，需要分层设计
**决策**: 
1. 采用4层架构：MCP接口层(server.py) → API业务逻辑层(api/) → 核心操作层(core/) → 工具层(utils/)
2. 使用FastMCP框架，确保与OpenAI MCP标准兼容
3. 统一错误处理机制，Excel操作失败时提供详细错误信息
**结果**: ✅ 建立清晰可扩展的架构，支持游戏开发常用Excel操作

## D002: 2025-12-05 数据类型统一化
**需求**: Excel数据类型标准化处理
**问题**: Excel读取的数据类型不统一，影响后续处理
**根因**: Python openpyxl库对Excel数据类型的解析不一致
**决策**: 
1. 统一数据类型映射规则：数字→int/float，文本→str，日期→datetime，空值→None
2. 添加数据类型验证和转换工具
3. 在所有Excel操作函数中应用统一的数据类型处理
**结果**: ✅ 解决数据类型不一致问题，提高后续处理的可靠性
