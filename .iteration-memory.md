### Round 12 — 多语句SQL + INSERT保留字修复 + ROLLUP/CUBE验证 ✅
**日期**: 2026-04-13  
**主题**: FROM子查询、多语句SQL、LIMIT行为、INSERT引号转义、相关子查询、ROLLUP探索、游戏配置场景  
**测试结果**: 42/42 通过 | 回归: 872/872 通过  

**新增功能**:
1. **多语句 SQL 支持** (`_execute_multi_statement`): 分号分隔的多条语句自动拆分执行，支持 SELECT/UPDATE/INSERT/DELETE 混合
   - 智能拆分：尊重字符串内的分号（单引号/双引号内不拆分）
   - 合并结果返回：所有 SELECT 的行数据合并，写入操作统计影响行数
   - 新增 `_execute_single_select` 辅助方法供多语句内部调用
   
2. **MySQL 保留字自动引用** (`_preprocess_reserved_words`): 确认仅4个保留字导致 ParseError:
   - `KEY` — 最常见（本地化表主键）
   - `INDEX`, `CHECK`, `CONSTRAINT`
   - 状态机逐字符分析，跳过字符串/反引号/数字字面量
   - 集成到 SELECT / INSERT / UPDATE / DELETE 四条路径

**Bug 修复**:
| Bug | 描述 | 原因 | 修复 |
|-----|------|------|------|
| 保留字ParseError | `Key`, `INDEX`, `CHECK`, `CONSTRAINT` 作为列名导致 sqlglot ParseError | MySQL 方言解析器将这些视为保留关键字 | `_preprocess_reserved_words()` 自动加反引号，状态机避免误替换字符串内内容 |
| 多语句只执行首条 | `SELECT a; SELECT b;` 只执行第一条 | `sqlglot.parse_one()` 只返回第一个语句 | 新增 `_execute_multi_statement()` 检测中间分号并拆分执行 |

**意外发现**:
- **ROLLUP/CUBE/GROUPING SETS 全部可用！** 原本预期失败但实际通过了。pandas 后端已支持这些 GROUP BY 扩展。
- 测试脚本 D1-D7 路由错误（is_update=True 导致 INSERT 被发给 UPDATE 函数）— 已修正为 is_update=False

**代码变更**:
- `advanced_sql_query.py`: +170 行（_execute_multi_statement + _execute_single_select）
- `test_round12.py`: 42 个测试用例（A-G 共8组）

## 已知限制（无需修复）
1. **无FROM标量SELECT**: `SELECT 1+1` 不支持（Excel查询必须有FROM）— 这是设计决策
2. **尾部分号**: SQL末尾的分号不会被当作多语句（这是正确行为）
3. **超长文本截断**: Excel单元格有字符限制（~32K或~50K取决于格式）

## 下轮建议 (Round 13)
1. **事务支持探索**: 多语句目前是逐条独立执行的，没有事务回滚能力。可考虑添加简单的事务包装
2. **CTE (WITH 子句) 深度测试**: Round 12 的子查询都通过了，可以专门测 CTE 的各种变体
3. **性能基准测试**: 对大数据量表（600+行）的复杂查询（JOIN+子查询+窗口函数组合）做性能 profiling
4. **INSERT RETURNING / OUTPUT**: 测试 INSERT 后是否需要返回插入的数据
5. **CREATE TABLE AS SELECT**: 探索是否支持从查询结果创建新工作表
