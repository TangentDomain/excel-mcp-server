### 发现的 Bug (2个，全部已修复)

#### Bug 1: NTILE/GROUP BY 列丢失 ✅ 已修复
**问题描述**: 当同时使用窗口函数和 GROUP BY 时，窗口函数的 ORDER BY 引用的非分组列在 GROUP BY 后丢失
```sql
-- 失败: Price 列在 GROUP BY 后不存在
SELECT Category, NTILE(4) OVER (ORDER BY Price) AS Tile 
FROM Products GROUP BY Category
```
**根本原因**: 
- 执行顺序是: GROUP BY → 窗口函数
- GROUP BY 后只保留分组列，Price 被丢弃
- 窗口函数执行时找不到 Price 列

**修复方案** (3处修改):
1. **Line 3690-3705**: 在 GROUP BY 前检测是否有窗口函数，如果有则先计算窗口函数
2. **Line 3720-3722**: 如果已预计算过窗口函数，跳过 GROUP BY 后的重复计算
3. **Line 6298-6313**: 在无聚合函数的早期返回路径中，包含预计算的窗口函数列
4. **Line 6407**: 修复 `hasattr(original_expr, "name")` 检查，确保空 name 不匹配（Window 的 name 为空字符串）
5. **Line 6414-6420**: 在有聚合函数时，处理 Window 表达式从 grouped 取 first

#### Bug 2: UPDATE CASE WHEN + numpy.uint8 ✅ 已修复 (Round 13 继承)
**问题描述**: UPDATE 语句中使用 CASE WHEN 时，如果目标列为 numpy.uint8 类型，会报 `'numpy.uint8' object has no attribute 'get'` 错误
```sql
UPDATE Products SET Price = CASE WHEN Category = 'Weapon' THEN Price * 1.1 ELSE Price END
```
**根本原因**: 
- Line 8536: `_evaluate_case_expression(expr, df.iloc[row_idx])` 使用位置参数
- `df.iloc[row_idx]` 是 Series，被绑定到 `df` 参数而非 `row` 参数
- 导致 `row=None`，触发向量化模式
- 向量化模式对 Series 的每个元素调用 `_get_row_value`
- numpy 标量没有 `.get()` 方法

**修复方案**:
- Line 8536: 改为 `_evaluate_case_expression(expr, None, row=df.iloc[row_idx])`
- 使用关键字参数确保 Series 正确绑定到 `row` 参数
