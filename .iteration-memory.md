### 🔴🟢 P0 Bug 发现与修复!

#### P0-3 [R27发现并修复]: UPDATE整数溢出截断 (uint8) 🚨→✅
**严重程度**: **CRITICAL 数据损坏** — UPDATE语句在混合类型Sheet中写入整数值>255时被截断

**复现条件**:
```sql
-- Sheet结构: ID(int), V(int), F(float) ← 必须有float列!
UPDATE sheet SET V = 999 WHERE ID = 1
-- 结果: V = 231 (即 999 % 256) ❌
```

**根本原因分析 (双重截断)**:
1. **第一重截断 (line ~7953 类型转换)**:
   - pandas将小整数列推断为 `uint8` dtype
   - `isinstance(np.uint8(10), (int, float))` 返回 False (numpy>=2.x行为变化)
   - 走入 `type(old_val)(new_val)` 分支 → `np.uint8(999)` = **231**
2. **第二重截断 (line ~7969 df.at赋值)**:
   - 即使绕过第一重, `df.at[idx, col] = 999` 对 uint8 列抛出 `TypeError`
   - pandas 内部 `LossySetitemError` 继承自 TypeError

**修复方案**: 两处修复缺一不可
1. 扩展 isinstance 检查覆盖 `np.integer` / `np.floating`
2. df.at 赋值加 try/except 跳过 (中间状态不影响最终结果)

**影响范围**: 所有包含混合 int+float 列的 Excel 文件执行 UPDATE 且值 > 255 时

##### P1-3 [R26新发现]: CTE 表别名导致列名前缀污染
**问题**: CTE 中使用表别名时，输出列名携带表别名前缀
```sql
WITH PlayerStats AS (
    SELECT p.GuildID, COUNT(*) as cnt  -- p 是 Players 的别名
    FROM Players p GROUP BY p.GuildID
)
SELECT ... FROM Guilds g JOIN PlayerStats ps ON g.GuildID = ps.GuildID
-- 报错: 右表 'PlayerStats' 没有列 'GuildID'.可用列: ['p.GuildID', 'cnt', 'avg_lvl']
```
**根本原因**: CTE 输出列名保留了源表的 `alias.column` 格式，未剥离前缀
**影响范围**: CTE + JOIN 组合查询中，当 CTE 源查询使用了表别名时
**优先级**: P1（有 workaround：不用表别名或手动指定完整列名）

### 数据格式说明（供后续测试参考）
> ⚠️ API 返回的 data 字段格式为 `[[header_row], [data_row1], [data_row2], ...]`
> 即第一行是列头（字符串），后续行是数据。测试断言时需注意：
> - `LIMIT 3` 返回 4 行（1 header + 3 data）
> - `COUNT(*)` 返回 `[['cnt'], [数值]]`（header + value）
> - 空表 `SELECT *` 返回 `[['col1','col2']]`（仅 header，0 data rows）

#### Bug R16-B1: IgnoreNulls/RespectNulls 包装节点未解包 ✅ 已修复

**根本原因**: sqlglot 将 `FUNC(x) IGNORE NULLS` 解析为 `IgnoreNulls(Func(x))` AST 节点
**修复方案**: 在 `_compute_window_function` 中添加 while 循环解包 `IgnoreNulls`/`RespectNulls` 节点 (~line 2833)

#### Bug R16-B2: UNION 括号 SELECT 导致 Subquery 解析失败 ✅ 已修复
**问题描述**: `(SELECT ...) UNION (SELECT ...)` 带括号的 UNION 查询报"未找到有效的 SELECT 语句"
**修复方案**: 在 `_extract_selects()` 中添加 `exp.Subquery` 分支 (~line 2563)

#### Bug R16-B3: UNION + ORDER BY 触发 _cn_to_en_map AttributeError ✅ 已修复
**问题描述**: 任何 UNION 查询加 ORDER BY 子句都报 `'AdvancedSQLQueryEngine' object has no attribute '_cn_to_en_map'`
**修复方案**: Line 2524 改用 `getattr(self, '_cn_to_en_map', {})`
