### 发现的 Bug (4个，全部已修复)

#### Bug R15-B1: JOIN 列映射在窗口函数中丢失 ✅ 已修复
**问题描述**: `JOIN + 窗口函数` 场景下，PARTITION BY / ORDER BY 中的 `table.col` 格式列名无法解析
```sql
-- 失败: p.Category 在 PARTITION BY 中无法解析
SELECT p.Name, ROW_NUMBER() OVER (PARTITION BY p.Category ORDER BY s.SaleDate) 
FROM Products p INNER JOIN Sales s ON p.ID = s.ProductID
```
**根本原因**: `resolve_col_name()` 和 `_compute_window_aggregate()` 只做裸列名匹配，不处理 `table.col` 或 `table_col` 格式
**修复方案**: 
- Line ~2854: `resolve_col_name()` 添加 `"table.col"` 和 `"table_col"` 后缀匹配
- Line ~3124: `_compute_window_aggregate()` 添加同样的 JOIN 列映射 + 后缀匹配逻辑

#### Bug R15-B2: LEAD/LAG 硬编码 dtype=float ✅ 已修复
**问题描述**: LEAD/LAG 对字符串列使用默认值时，因硬编码 `dtype=float` 导致 TypeError
```sql
-- 失败: Name 是字符串列，LEAD 默认值 'N/A' 赋值到 float Series 报错
SELECT Name, LEAD(Name, 1, 'N/A') OVER (ORDER BY Name) FROM Products
```
**根本原因**: `_compute_lag`/`_compute_lead` 创建结果 Series 时硬编码 `dtype=float`
**修复方案**: 推断目标列的实际 dtype: `target_dtype = df[target_col].dtype`

#### Bug R15-B3/B3b: 内层聚合表达式反向查找 ✅ 已修复
**问题描述**: `LEAD(MAX(Name))` / `LAG(MAX(Price))` 等嵌套场景，内层聚合 AST 节点的 `.name` 返回空字符串
```sql
-- 失败: MAX(Name) 的 .name 为空，导致 KeyError
SELECT Name, LEAD(MAX(Name), 1) OVER (ORDER BY Name) FROM Products GROUP BY Name
```
**根本原因**: sqlglot 的 Max/Min/Sum 等 AST 节点没有 `.name` 属性（只有 Column 节点有）
**修复方案**: 
- 通过 `select_alias_map` 反向查找匹配的别名
- 新增 **R15-B3b**: 当 alias_map 也无有用映射时，递归遍历 AST 树直到找到 Column 节点

#### Bug R15-B1c: GROUP BY 后窗口聚合函数列解析 ✅ 已修复（核心阻塞Bug）
**问题描述**: `JOIN + GROUP BY + 多窗口函数` 组合中，窗口聚合函数引用的原始列在 GROUP 后不存在
```sql
-- 失败: AVG(s.Quantity) 在 GROUP BY 后执行，s.Quantity 已被聚合为 TotalSold
SELECT p.Name, p.Category, SUM(s.Quantity) AS TotalSold,
       ROW_NUMBER() OVER (ORDER BY SUM(s.Quantity) DESC) AS RowNum,
       AVG(s.Quantity) OVER (PARTITION BY p.Category) AS CatAvg
FROM Products p INNER JOIN Sales s ON p.ID = s.ProductID
GROUP BY p.Name, p.Category
```
**根因分析**:
1. 预计算路径（GROUP BY 前）对 `ROW_NUMBER() OVER (ORDER BY SUM(s.Quantity))` 失败（SUM 尚未计算）
2. 异常被静默捕获 → 全部窗口函数回退到 GROUP BY 后执行
3. GROUP BY 后 DataFrame 只有 `[p.Name, p.Category, TotalSold, RowNum, CatAvg]`
4. `AVG(s.Quantity)` 的 `s.Quantity` 不在任何映射中 → KeyError

**修复方案**:
- 将 `select_alias_map` 参数传递给 `_compute_window_aggregate()`
- 在列解析末尾添加通过 `select_alias_map` 反向查找的逻辑
- 用正则从聚合表达式中提取内部列名进行匹配: `SUM(s.Quantity)` → 提取 `s.Quantity` → 匹配 `TotalSold`

## 下轮建议 (Round 16)

### 高优先级
1. **窗口函数预计算优化**: 从"全有或全无"改为逐个窗口函数分别尝试预计算
2. **NTH_VALUE 窗口函数测试**: 目前覆盖较少
3. **PERCENT_RANK / CUME_DIST / NTILE 与复杂查询组合**

### 中优先级
4. **UPDATE 语句中的子查询**: UPDATE SET col = (SELECT ...) 
5. **DELETE + JOIN**: DELETE ... FROM t1 INNER JOIN t2 ON ...
6. **UNION / UNION ALL + 窗口函数**

### 低优先级
7. **性能基准**: 10000+ 行数据下窗口函数性能
8. **更多边缘情况**: 全 NULL 分区、单行分组、空字符串与 NULL 混合
