### 发现的问题

#### Bug #2: 列名大小写敏感 🔴 已修复
- **严重程度**: 中
- **描述**: SQL中列名必须精确匹配Excel表头的大小写，如 `SELECT RATE FROM ...` 无法匹配名为 `Rate` 的列。SQL 标准规定未引用标识符(identifier)应大小写不敏感。
- **影响范围**: SELECT、WHERE、ORDER BY、UPDATE SET 所有涉及列名解析的路径
- **修复**: 新增 `_find_column_name()` 方法实现大小写不敏感列名查找；修复了 `_evaluate_math_expression`(数学表达式)、`_expression_to_column_reference`(WHERE条件)、`_resolve_order_column`(ORDER BY)、SELECT表达式处理、UPDATE SET 共 5 个关键路径的列名匹配逻辑。
- **Commit**: `ba6ea3b`
- **回归**: 872 测试全通过

#### 非Bug说明
- E1 (`三层函数嵌套`) 初始失败原因: SQL 中使用 `RATE`(全大写) 但列名为 `Rate`，属于 Bug #2 的表现。修复 Bug #2 后自动通过。
