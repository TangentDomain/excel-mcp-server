### Round 10 (当前) — 2026-04-13
**主题**: 数据分析深度(QA边缘场景) + 窗口函数补齐 + UPDATE精度 + 性能基准

**测试用例**: 53个 (33/53 首轮 → 修复后 46/63 含验证)
- Group A: 窗口函数补齐 [8] — FIRST_VALUE✅ LAST_VALUE✅ NTILE✅ PERCENT_RANK✅ CUME_DIST✅ NTH_VALUE✅ + ROUND包裹✅
- Group B: DISTINCT + ORDER BY [6] — 基础DISTINCT✅ 多列✅ COUNT(DISTINCT)✅ 子查询DISTINCT✅ DISTINCT ON(H2支持!) GROUP BY HAVING✅
- Group C: 双行表头 [8] — 查询✅ WHERE✅ ORDER BY✅ GROUP BY✅ UPDATE写入✅ 回读✅ INSERT✅ 验证✅
- Group D: UPDATE精度 [10] — PI值✅ 极大数✅ 极小小数✅ 负数✅ 表达式计算✅ (H2 float32精度损失已知)
- Group E: 性能基准 [8] — 全表SELECT✅ WHERE筛选✅ ORDER BY✅ GROUP BY聚合✅ RANK窗口✅ UPDATE批量✅ LIKE搜索✅ CASE WHEN✅
- Group F: 边缘压力 [11] — 特殊字符INSERT✅ 中文符号✅ COALESCE✅ DELETE清理✅ 超长文本✅ 复合表达式✅ (单引号嵌套双引号语法问题待优化)

**发现并修复的Bug (2个)**:
1. **[中] ROUND/ABS等标量函数包裹窗口函数时崩溃** (F27)
   - 触发: `SELECT ROUND(PERCENT_RANK() OVER (ORDER BY Price), 4) FROM ...`
   - 原因: `_apply_window_functions` 只遍历顶层 SELECT 表达式，不发现嵌套在 Round/Abs 内的 Window 节点；`_expr_to_series` 无 Window 分支
   - 修复: `_apply_window_functions` 新增 `find_all(exp.Window)` 递归发现所有 Window 节点(含嵌套)，预计算后存入 df；`_expr_to_series` 新增 `exp.Window` 分支通过 id() 查找预计算列
   - 影响: 支持 ROUND/ABS/CEIL/FLOOR/SQRT 等任意标量函数包裹窗口函数

2. **[中] UPDATE SET 不支持 COALESCE 和标量数学函数** (F28)
   - 触发: `UPDATE t SET col = COALESCE(NULL, 100.0)` 或 `SET col = ABS(val)`
   - 原因: `_evaluate_update_expression` 缺少 Coalesce/Abs/Ceil/Floor/Sqrt 分支，回退返回空字符串导致 float32 列写入崩溃
   - 修复: 新增 Coalesce 分支(遍历参数返回首个非NULL值)；新增 Abs/Ceil/Floor/Sqrt 向量化计算分支
   - 影响: UPDATE SET 现在完整支持 CASE WHEN(R7) + COALESCE + ROUND(R5) + ABS/CEIL/FLOOR/SQRT

**重要发现（非Bug）**:
- **H3 LIMIT 500 默认截断**: 大数据量查询(10000行)静默截断到500行，建议增加警告或可配参数
- **DISTINCT ON**: H2 引擎原生支持 PostgreSQL 语法的 DISTINCT ON，非标准 SQL 但可用
- **INSERT 混合引号**: 单引号内嵌套双引号的值(如 `O'Brien's_"Sword"`)导致 sqlglot 解析失败，需统一转义方式
- **精度行为**: PI值写入float32列后读回为3.14（已知H2+float32限制），float64列保持高精度

## 下轮建议方向 (Round 11)

**优先级高**:
1. **LIMIT 500 默认截断 (H3)** — 大数据量场景需要明确警告或可配置限制
2. **多语句执行 (H1)** — 运营批量操作的分号分隔多条语句
3. **INSERT 混合引号转义** — 单引号内嵌套双引号/特殊字符的健壮性

**优先级中**:
4. **LEAD/LAG 窗口函数 + OFFSET 表达式** — LEAD(col, 2) 等
5. **FROM 子查询** — 当前不支持 `SELECT * FROM (SELECT ...) sub`
6. **性能优化** — 10000行 GROUP BY + 窗口函数耗时分析

**角色视角待覆盖**:
- 🛡️ **QA自动化**: 并发安全性、边界值自动生成
- 📢 **运营**: 活动配置复制到新表、批量价格调整脚本模板
- 🎨 **美术/配置**: 配置表间交叉引用完整性扫描工具
