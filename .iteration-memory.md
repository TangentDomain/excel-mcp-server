### 发现的 Bug + 修复

#### Bug 1: `_in_to_pandas` 子查询结果丢失第一行数据 🔴 Critical
**文件**: `advanced_sql_query.py:5506`  
**现象**: `UPDATE ... WHERE ID IN (SELECT ...)` 只匹配部分目标行  
**根因**: `sub_result.iloc[1:, 0]` 错误地跳过了第一行数据。注释声称"排除表头行"，但 `_execute_subquery` 返回的 DataFrame **不包含表头行**（columns 属性就是列名，数据从 iloc[0] 开始）  
**影响范围**: 所有使用 IN 子查询的 UPDATE/DELETE 操作  
**修复**: `iloc[1:, 0]` → `iloc[:, 0]`  
**验证**: IN 子查询 UPDATE 从 affected=1 修正为 affected=2 ✅

#### Bug 2: `_evaluate_condition_for_row` 不支持 IN 子查询 🟡 Medium
**文件**: `advanced_sql_query.py:5822-5825`  
**现象**: 当 pandas query 路径失败、回退到逐行过滤时，`IN (SELECT ...)` 无法正确处理  
**根因**: `exp.In` 分支只处理了字面量列表 `IN (1,2,3)`，没有检测 `expressions[0]` 是否为 Subquery 节点  
**修复**: 增加 Subquery/Select 类型检测分支，执行子查询后做类型兼容的成员关系检查  
**验证**: CTE + UPDATE/DELETE 回退路径正常工作 ✅

#### 🟡 P1 — 应该修复
1. **NTILE 在 GROUP BY 后泄漏额外列** — `SELECT Category, NTILE(4) OVER (ORDER BY Price) FROM Products GROUP BY Category` 返回 3 列而非 2 列
2. **窗口函数 + COUNT(*) GROUP BY 冲突** — `SELECT COUNT(*), ROW_NUMBER() OVER() FROM t GROUP BY Type` 中 COUNT(*) 被 window post-processing 错误解析
3. **CROSS JOIN 行数差异** — `SELECT * FROM A, B` 返回 9 行 vs `CROSS JOIN` 返回 3 行
4. **ORDER BY alias + LIMIT 交互异常** — `ORDER BY c LIMIT 1` 在列别名 'c' 存在时返回错误行

#### 🟢 P2 — 可以优化
5. **DISTINCT + 窗口函数** — `SELECT DISTINCT Level, ROW_NUMBER() OVER (PARTITION BY Level)` 抛出 "cannot reindex on an axis with duplicate labels"
6. **多窗口函数同一 PARTITION BY 不同 ORDER BY** — 只有最后一个生效
7. **FROM 子查询中的 LIMIT/OFFSET** — 嵌套派生表中的 LIMIT 有时不生效
8. **CASE WHEN + 聚合函数嵌套** — 部分场景返回空字符串

#### ⚪ P3 — 低优先级 / 设计如此

10. **UPDATE/DELETE 小规模写入性能**: <100 行更新需要 2-5 秒（openpyxl I/O 开销，可接受）
