### 🔴 重大发现

#### P0-1: script_runner 可执行任意系统命令（root权限）🚨
**严重程度**: **CRITICAL** — excel_run_python 工具提供完整的 Python exec() 环境，可：
- 导入 os/subprocess 模块 ✅
- 读取任意文件 (/etc/passwd) ✅
- 执行 shell 命令 (`id`, `echo hacked`) ✅
- 以 root 权限运行一切
**影响范围**: 所有使用 excel_run_python 的场景
**建议修复**: 沙箱隔离、禁用危险模块(import hook)、或明确文档警告

#### P0-2: SELECT 支持分号多语句执行 🚨
**严重程度**: **HIGH** — `SELECT ...; SELECT ...` 或 `SELECT...; DROP...` 格式的多语句 SQL 可以成功执行
- UPDATE/INSERT/DELETE 路径已正确拒绝多语句 ✅
- 但 SELECT 路径允许分号分隔的多语句 ❌
- 实测: `SELECT COUNT(*) FROM t1; SELECT COUNT(*) FROM t2` → "2/2 条成功"
**影响范围**: 所有 excel_query 调用
**建议修复**: 在 SQL 解析入口统一拒绝含分号的多语句

#### P1-1: ExcelOperations 构造函数设计问题
**问题**: `ExcelOperations(file_path)` 不接受参数（构造函数签名 `(self, /, *args, **kwargs)`），但 script_runner 将其暴露给用户时暗示可以实例化
**影响**: 用户在 script_runner 中写 `ops = ExcelOperations(file_path)` 会 TypeError
**现状**: 正确用法是 `ExcelOperations.method_name(file_path, ...)` （类方法调用风格）

#### P1-2: SQL 注释符 `--` 被正常执行
**问题**: `SELECT * FROM t WHERE ID = 1 -- 注释` 成功执行并返回结果
**影响**: 可能与其他注入向量组合利用
**优先级**: 中等（单独无害，但降低整体安全性）

### 发现的 Bug (3个，全部已修复)

#### Bug R16-B1: IgnoreNulls/RespectNulls 包装节点未解包 ✅ 已修复
**问题描述**: `NTH_VALUE(Score, 1) IGNORE NULLS OVER (...)` 等 SQL 标准语法报"不支持的窗口函数: IgnoreNulls"
**根本原因**: sqlglot 将 `FUNC(x) IGNORE NULLS` 解析为 `IgnoreNulls(Func(x))` AST 节点
**修复方案**: 在 `_compute_window_function` 中添加 while 循环解包 `IgnoreNulls`/`RespectNulls` 节点 (~line 2833)

#### Bug R16-B2: UNION 括号 SELECT 导致 Subquery 解析失败 ✅ 已修复
**问题描述**: `(SELECT ...) UNION (SELECT ...)` 带括号的 UNION 查询报"未找到有效的 SELECT 语句"
**修复方案**: 在 `_extract_selects()` 中添加 `exp.Subquery` 分支 (~line 2563)

#### Bug R16-B3: UNION + ORDER BY 触发 _cn_to_en_map AttributeError ✅ 已修复
**问题描述**: 任何 UNION 查询加 ORDER BY 子句都报 `'AdvancedSQLQueryEngine' object has no attribute '_cn_to_en_map'`
**修复方案**: Line 2524 改用 `getattr(self, '_cn_to_en_map', {})`

## 已知问题清单

### P0 — 🔴 必须修复
1. **[R25] script_runner 任意代码执行**: `excel_run_python` 工具提供完整 Python exec() 环境，可执行任意系统命令（root权限）。需沙箱隔离或禁用危险模块。
2. **[R25] SELECT 分号多语句注入**: `SELECT ...; SELECT ...` 可成功执行多条语句。UPDATE/INSERT/DELETE 已正确拦截，但 SELECT 路径遗漏。

### P1 — 🟡 应尽快修复
3. **[R25] ExcelOperations 构造函数设计缺陷**: 不接受 file_path 参数，但 script_runner 暴露给用户时暗示可实例化。用户代码会 TypeError。
4. **[R25] SQL 注释符 `--` 未处理**: 尾随注释被正常执行，可能与其他注入向量组合利用。

### P2 — 无

### P3 - 预期限制
- COALESCE/复杂表达式在 PARTITION BY 中不支持（需要表达式求值后作为分区列）
- 窗口函数预计算是"全有或全无"模式——一个失败则全部回退到 GROUP BY 后执行
- 窗口函数无别名时可能触发 'col_N' 列名找不到的错误（建议使用 AS 别名）
