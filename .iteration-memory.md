- Group C: 运营批量操作 [5] ✅ — Epic涨20%、Common打五折、INSERT活动、UPDATE状态、DELETE清理

- Group E: 数据分析统计 [4] ✅ — 稀有度经济统计、商店差价分析、掉落期望值、HAVING筛选
- Group F: 边缘场景压力 [5] ✅(3/5→修正后5/5) — 负数价格、零掉率、COALESCE NULL、INSERT特殊字符、DELETE清理

**发现并修复的Bug (1个)**:
1. **[中] JOIN ON 条件不支持表达式拼接 (CONCAT/数学表达式)**
   - 触发条件: ON 子句使用 `CONCAT('prefix', col, '_suffix') = other.col` 或 `col * 1 = other.col`
   - 原因: `_parse_join_on_condition` 的 `resolve_column` 对非 Column 表达式返回字符串表示，被误当作列名传给 pandas merge
   - 修复: 增加 `is_simple_column` 标记，当 EQ 任一侧不是简单列引用时路由到 cross join + row filter 非等值连接路径
   - 影响: LEFT/RIGHT/FULL OUTER JOIN 的 ON 条件现在支持 CONCAT()、数学运算、函数调用等表达式
   - 注意: MySQL 模式下 `||` 是逻辑 OR 非字符串拼接，需用 CONCAT() 函数

**重要发现（非Bug，使用方式问题）**:
- **D2 SQL保留字 `Key`**: 不加引号会报语法错误，用反引号 `` `Key` `` 可正常工作 → 非Bug
- **F4 INSERT特殊字符**: 单引号值内嵌套单引号需用双引号包裹或 SQL 标准 `''` 转义 → 非Bug

**Commit**: `5b9b06c` (v1.9.5)
