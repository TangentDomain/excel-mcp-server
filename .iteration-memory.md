### 发现的 Bug (3个，全部已修复)

#### Bug R16-B1: IgnoreNulls/RespectNulls 包装节点未解包 ✅ 已修复
**问题描述**: `NTH_VALUE(Score, 1) IGNORE NULLS OVER (...)` 等 SQL 标准语法报"不支持的窗口函数: IgnoreNulls"
```sql
-- 失败: sqlglot 解析为 IgnoreNulls(NthValue(...))，分发器不识别
SELECT Name, Score, NTH_VALUE(Score, 1) IGNORE NULLS OVER (ORDER BY Score DESC) FROM Employees
```
**根本原因**: sqlglot 将 `FUNC(x) IGNORE NULLS` 解析为 `IgnoreNulls(Func(x))` AST 节点，`_compute_window_function` 直接取 `type(window_expr.this).__name__` 得到 `"IgnoreNulls"` 而非实际函数名
**修复方案**: 
- Line ~2833: 在 `_compute_window_function` 中添加 while 循环解包 `IgnoreNulls`/`RespectNulls` 节点
- 解包后用 `window_expr.set("this", actual_func)` 替换，确保下游处理器正常工作

#### Bug R16-B2: UNION 括号 SELECT 导致 Subquery 解析失败 ✅ 已修复
**问题描述**: `(SELECT ...) UNION (SELECT ...)` 带括号的 UNION 查询报"未找到有效的 SELECT 语句"
```sql
-- 失败: 左右分支被解析为 Subquery 节点而非 Select
(SELECT Name, Salary FROM E WHERE S > 15000) UNION (SELECT ItemName, Price FROM I WHERE P > 100)
```
**根本原因**: `_extract_selects()` 递归函数只处理 `exp.Select` 和 `exp.Union`，不处理 `exp.Subquery`。带括号的 SELECT 被 sqlglot 解析为 Subquery > Select
**修复方案**: 
- Line ~2563: 在 `_extract_selects()` 中添加 `exp.Subquery` 分支，递归解包 `node.this`

#### Bug R16-B3: UNION + ORDER BY 触发 _cn_to_en_map AttributeError ✅ 已修复
**问题描述**: 任何 UNION 查询加 ORDER BY 子句都报 `'AdvancedSQLQueryEngine' object has no attribute '_cn_to_en_map'`
```sql
-- 失败: _apply_union_order_by 引用 self._cn_to_en_map 但该属性从未初始化
SELECT a FROM t1 UNION SELECT b FROM t2 ORDER BY 1
```
**根本原因**: `_apply_union_order_by()` Line 2524 直接访问 `self._cn_to_en_map`，但该属性仅在部分代码路径中初始化（表头解析时），UNION 查询路径不经过表头解析
**修复方案**: 
- Line 2524: 改用 `getattr(self, '_cn_to_en_map', {})` 提供默认空字典
