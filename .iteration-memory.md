### 发现的问题

#### 🔴 已修复 (commit `14e24bf`)
| ID | 描述 | 严重程度 | 根因 |
|----|------|---------|------|
| F2 | 窗口函数在混合类型object列上sort_values崩溃 | 🔴高 | 部分列INSERT写入空串→object dtype→int+str比较TypeError |

**F2 修复详情**: 在 `_apply_window_functions` 和 `_compute_window_function` 两处 `df.sort_values()` 前添加 `pd.to_numeric(errors='coerce')`; 所有 `.astype(int)` → `.astype('Int64')` 支持 NaN。

#### 🟡 已知限制 (不修，记录)
| ID | 描述 | 原因 |
|----|------|------|
| C2b | 窗口函数 ORDER BY 中 COALESCE 别名不可用 | 窗口解析器只支持原始列名，不支持表达式别名引用 |
| F5 | 关联标量子查询(外层列引用内层)失败 | 子查询执行上下文无法访问外层查询的列别名；需架构级改造 |

### 关键发现
1. **空表查询返回格式**: `success=True, data=[header_row]`（len=1），不是空列表。验证空结果应检查 `len(data) == 1`
2. **双行表头完全支持**: SELECT/WHERE/UPDATE 均正常工作
3. **UNION ALL / INTERSECT / EXCEPT 全部通过**
4. **CROSS JOIN 笛卡尔积正常工作**
5. **大数据性能**: 1200行 GROUP BY + 窗口函数均在 1.2s 内完成
6. **字符串/数学函数全覆盖**: CONCAT, REPLACE, UPPER, LENGTH, ABS, CEIL, FLOOR, POWER, SQRT, MOD
