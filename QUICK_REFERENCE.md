## 🎯 最佳实践指南

### 💡 使用技巧

#### 🎯 表达方式优化
- **简洁指令**：直接说操作目标，AI会自动选择合适工具
- **上下文明确**：提供足够的表名和字段名信息
- **批量操作**：多个相同类型操作可以一次性完成
- **分步验证**：复杂操作可以分步骤执行并验证结果

#### 📋 数据结构规范
```bash
# 工作表命名规范
技能表: skills, 技能表, skill_table
装备表: items, 装备表, equipment
角色表: characters, 角色表, character
怪物表: monsters, 怪物表, enemy

# 字段命名规范
ID字段: id, ID, 编号, ID
名称字段: name, 名称, name, title
数值字段: value, 数值, amount, count
```

#### ⚡ 性能优化建议
- **小文件**(<1万行)：直接处理，无需特殊优化
- **中文件**(1-10万行)：启用流式处理，批量操作
- **大文件**(>10万行)：分块处理，避免全量加载

### 🚫 避免的常见错误

#### ❌ 指令不明确
- ❌ "处理技能表"
- ✅ "技能表中法师技能的MP消耗减少10%，并按新消耗值排序"

#### ❌ 缺少关键信息
- ❌ "给装备加属性"
- ✅ "为装备表中所有稀有度=史诗的装备增加攻击力+5，防御+3"

#### ❌ 数据类型错误
- ❌ "把攻击力的文本10转换为数字"
- ✅ "将攻击力列中的文本数字转换为数值格式"

## 🔧 故障排除

### 🆘 常见问题快速解决

#### 数据格式问题
```bash
# 数字变文本问题:
"将文本格式的数字转换为数值格式"
"检查单元格格式,设置为常规或数值"
```

#### 跨表关联失败
```bash
# JOIN查询失败:
"检查关联字段的数据类型是否一致"
"确认关联值在两个表中都存在"
"使用模糊匹配查找可能的不一致"
```

#### 性能问题
```bash
# 大文件处理优化:
"处理后及时清理内存,避免内存泄漏"
"使用分页查询,每次只加载部分数据"
"关闭不必要的Excel功能,减少内存占用"
```

### 📞 获取帮助

1. **查看日志**：检查 `.excel_mcp_logs/` 目录下的错误日志
2. **简化问题**：先用小文件测试，复现问题后再处理大文件
3. **版本确认**：运行 `excel-mcp-server-fastmcp --version` 确认版本
4. **提交Issue**：[GitHub Issues](https://github.com/TangentDomain/excel-mcp-server/issues/new)
