## 分析发现

### 1. `_in_to_pandas` 方法分析 (3043-3071行)

**核心逻辑**:
```python
def _in_to_pandas(self, in_expr: exp.In, df: pd.DataFrame, negate: bool = False) -> str:
    left = self._expression_to_column_reference(in_expr.this, df)
    prefix = "~" if negate else ""

    # 子查询模式 (IN (SELECT ...))
    subquery = in_expr.args.get('query')
    if subquery and isinstance(subquery, exp.Subquery):
        sub_result = self._execute_subquery(subquery, self._current_worksheets)
        if len(sub_result.columns) > 0:
            sub_values = sub_result.iloc[:, 0].dropna().tolist()  # ⚠️ 问题点1
            values_str = ', '.join(repr(v) for v in sub_values)  # ⚠️ 问题点2
            return f"{prefix}{left}.isin([{values_str}])"
```

**问题点**:
1. **数据获取方式**: `sub_result.iloc[:, 0]` 直接获取第一列的所有行数据
2. **值序列化**: 使用 `repr(v)` 将每个值转换为Python字符串表示
3. **Query字符串构造**: 直接拼接生成pandas query表达式

### 4. pandas query字符串生成问题

**问题根因定位**:

在 `_in_to_pandas` 方法的3061行：
```python
values_str = ', '.join(repr(v) for v in sub_values)
```

**潜在问题**:

#### 问题A: repr()的副作用
- `repr()` 会将字符串值加上引号（单引号或双引号）
- 对于中文字符，`repr()` 可能产生 Unicode 转义序列
- 对于特殊字符（换行、引号等），`repr()` 会进行转义

#### 问题B: 值类型处理
- 如果子查询返回混合类型（字符串、数字、日期等）
- `repr()` 产生的字符串表示不一致
- 拼接到query字符串可能导致语法错误

#### 问题C: 列名与数据混淆
- 如果子查询只有一行，且DataFrame列名被误当作数据
- `iloc[:, 0]` 会先访问列名，然后访问数据行
- 但实际上pandas DataFrame的 `iloc[:, 0]` 只返回数据行，不包含列名

**验证分析**:
```python
# 假设子查询返回:
# DataFrame: {'id': [1, 2, 3]}
sub_values = sub_result.iloc[:, 0].dropna().tolist()
# 结果: [1, 2, 3]  # ✅ 正确，不包含列名

# 假设子查询返回:
# DataFrame: {'姓名': ['张三', '李四']}
sub_values = sub_result.iloc[:, 0].dropna().tolist()
# 结果: ['张三', '李四']  # ✅ 正确，不包含列名
```

## 根因总结

### 主要问题
**repr() 在处理中文字符和特殊字符时的转义行为导致pandas query字符串生成错误**

### 次要问题
1. 值类型未检查，直接拼接可能导致语法错误
2. 空值处理依赖于 `dropna()`，但如果所有值都是NaN，会导致空列表
3. 错误处理捕获所有异常，可能掩盖具体问题

### 不存在的问题
1. **子查询结果不包含表头行** - DataFrame结构正确，列名已分离
2. **数据获取方式正确** - `iloc[:, 0]` 只返回数据行，不包含列名

## 建议修复方案

### 方案A: 替换 repr() 为更安全的序列化
```python
# 对于字符串值，手动添加引号并转义
def safe_repr(v):
    if isinstance(v, str):
        # 手动转义特殊字符
        escaped = v.replace('\\', '\\\\').replace('"', '\\"').replace("'", "\\'")
        return f'"{escaped}"'
    elif pd.isna(v):
        return 'None'
    else:
        return str(v)

values_str = ', '.join(safe_repr(v) for v in sub_values)
```

### 方案B: 使用pandas内置的字符串表示
```python
# 让pandas处理字符串转义
import json
values_str = ', '.join(json.dumps(v, ensure_ascii=False) for v in sub_values)
```

### 方案C: 添加类型检查和错误处理
```python
if not sub_values:
    return f"{prefix}{left}.isin([])"

# 检查值类型一致性
value_types = set(type(v).__name__ for v in sub_values)
if len(value_types) > 1:
    raise ValueError(f"子查询返回的值类型不一致: {value_types}")

# 使用更安全的序列化
values_str = ', '.join(self._value_to_query(v) for v in sub_values)
```

## 结论

REQ-060 子查询IN问题的根因是 **`repr()` 函数在序列化子查询结果时的转义行为导致pandas query字符串生成错误**，特别是对于中文字符和特殊字符的处理不当。
